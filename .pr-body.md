## Summary

Implements the experiment gate checker (task P0-09): compute promotion decisions using hard gates and weighted ranking.

## Changes

- **`contracts/schema/experiment-decision.json`** — JSON Schema for experiment decision output
- **`services/eval-orchestrator/src/gate-checker.ts`** — Hard gate checks + `computeDecision` producing `ExperimentDecision`
- **`services/eval-orchestrator/src/score.ts`** — Weighted ranking score computation (`computeRankScore`, `computeRankComponents`)
- **`services/eval-orchestrator/src/types.ts`** — Shared type definitions
- **`services/eval-orchestrator/src/defaults.ts`** — Configurable default weights and thresholds (outside core logic)
- **`services/eval-orchestrator/tests/gate-checker.test.ts`** — 32 tests covering all definition-of-done criteria

## Hard Gates (from feedback-loop.md §6.1)

| Gate | Threshold |
|------|-----------|
| degenerate_rate | ≤ 3% |
| coherence | ≥ champion - 0.02 |
| correctness | ≥ champion - 0.01 |
| language_stability | ≥ 99% |
| p95_latency_ms | ≤ champion × 1.20 |
| safety_critical | 0 violations |

## Rank Score (from feedback-loop.md §5.3)

```
rank_score = 0.35×correctness + 0.20×coherence + 0.20×concept_adherence
           + 0.10×solve_rate_norm + 0.10×(1-degenerate_rate) + 0.05×latency_norm
```

Weights are configurable via `defaults.ts`, not hardcoded in logic.
